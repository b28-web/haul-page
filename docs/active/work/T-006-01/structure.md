# T-006-01 Structure: Content Resources

## Files Created

### Domain Module

**`lib/haul/content.ex`** — Replace existing implicit module path (currently `content/loader.ex` lives here)
- `use Ash.Domain`
- Register all five resources

### Resources

**`lib/haul/content/site_config.ex`**
- `use Ash.Resource` with `AshPostgres.DataLayer` + `AshPaperTrail.Resource`
- Tenant-scoped (`multitenancy strategy: :context`)
- Table: `site_configs`
- Attributes: id, business_name, phone, email, tagline, service_area, address, coupon_text, meta_description, primary_color, logo_url, timestamps
- Actions: defaults [:read], create :create_default (accept all fields), update :edit (accept all fields)
- Code interface: `define :current, action: :read`

**`lib/haul/content/service.ex`**
- `use Ash.Resource` with `AshPostgres.DataLayer` + `AshPaperTrail.Resource`
- Tenant-scoped
- Table: `services`
- Attributes: id, title, description, icon, sort_order, active, timestamps
- Actions: defaults [:read, :destroy], create :add, update :edit
- Preparations: sort by sort_order, filter active == true on :read

**`lib/haul/content/gallery_item.ex`**
- `use Ash.Resource` with `AshPostgres.DataLayer` + `AshPaperTrail.Resource`
- Tenant-scoped
- Table: `gallery_items`
- Attributes: id, before_image_url, after_image_url, caption, alt_text, sort_order, featured, active, timestamps
- Actions: defaults [:read, :destroy], create :add, update :edit

**`lib/haul/content/endorsement.ex`**
- `use Ash.Resource` with `AshPostgres.DataLayer` + `AshPaperTrail.Resource`
- Tenant-scoped
- Table: `endorsements`
- Attributes: id, customer_name, quote_text, star_rating (integer 1-5), source (EndorsementSource enum), date, featured, active, timestamps
- Relationships: `belongs_to :job, Haul.Operations.Job, allow_nil?: true`
- Actions: defaults [:read, :destroy], create :add, update :edit

**`lib/haul/content/endorsement/source.ex`**
- `use Ash.Type.Enum`
- Values: [:google, :yelp, :direct, :facebook]

**`lib/haul/content/page.ex`**
- `use Ash.Resource` with `AshPostgres.DataLayer` + `AshPaperTrail.Resource`
- Tenant-scoped
- Table: `pages`
- Attributes: id, slug, title, body, body_html, meta_description, published, published_at, timestamps
- Identities: unique_slug on [:slug]
- Actions: defaults [:read, :destroy], create :draft, update :edit (with body_html stub), update :publish, update :unpublish

## Module Boundaries

- `Haul.Content` is the only public API surface for content queries
- Resources are queried via `Ash.read!/2`, `Ash.get!/2`, `Ash.create!/2`, etc.
- No code_interface definitions except SiteConfig `:current` — keep the API surface small until admin UI needs drive expansion
- Cross-domain reference: `Endorsement -> Job` is a read-only relationship (no actions that traverse it)

## Ordering

1. Create enum type (Source) — no dependencies
2. Create domain module — must exist before resources reference it
3. Create resources (any order — they only reference the domain and the Job resource which already exists)
4. Update config.exs — must happen before compilation
5. Generate migrations — must happen after resources compile
6. Run migrations — must happen after generation
7. Write tests — after migrations so DB schema exists
8. Run tests — final verification
