# T-003-01 Structure: Job Resource

## New files

### `lib/haul/operations.ex`
Ash domain declaration. Registers `Haul.Operations.Job`.

```
defmodule Haul.Operations do
  use Ash.Domain
  resources -> Haul.Operations.Job
end
```

### `lib/haul/operations/job.ex`
Ash resource with AshStateMachine extension. ~80 lines.

Sections:
- `postgres` — table "jobs", repo Haul.Repo, multitenancy :context
- `state_machine` — initial_states [:lead], default_initial_state :lead, full state list
- `attributes` — id, customer_name, customer_phone, customer_email, address, item_description, preferred_dates ({:array, :date}), notes, timestamps
- `actions` — defaults [:read], create :create_from_online_booking accepting all public attrs

### `test/haul/operations/job_test.exs`
Tests for Job resource. ~80 lines.

Tests:
- Creates a job via :create_from_online_booking with valid attrs → state is :lead
- Requires customer_name (nil → error)
- Requires customer_phone (nil → error)
- Requires address (nil → error)
- Requires item_description (nil → error)
- customer_email is optional
- preferred_dates stores a list of dates
- notes is optional

Setup: create company → derive tenant schema → use as tenant context.

### `priv/repo/tenant_migrations/YYYYMMDD_create_operations.exs`
Generated by `mix ash_postgres.generate_migrations`. Creates `jobs` table in tenant schema with all columns.
