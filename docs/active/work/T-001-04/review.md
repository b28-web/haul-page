## Image Size: 278MB vs 100MB Target

The 100MB target is not achievable with the current dependency set on Debian:

- **Base image** (debian:bookworm-slim): ~75MB
- **Runtime packages** (openssl, libstdc++6, libncurses5, locales, ca-certificates): ~31MB
- **Release** (ERTS + all OTP apps + Ash ecosystem): ~81MB
- **Locale generation**: ~3MB

The release alone is 81MB. The Ash ecosystem (ash 13MB, ex_cldr 10MB, digital_token 4MB) accounts for ~27MB. These are required dependencies.

**Options to reduce further (not implemented, diminishing returns):**
1. Alpine Linux — saves ~40MB on base but risks musl/glibc NIF issues
2. Remove locale support — saves ~3MB but breaks ex_cldr
3. Custom OTP build without unused apps — high effort, saves ~5-10MB
4. Strip more aggressively — `strip_beams` is already default in mix release

**Recommendation:** Accept 278MB for now. This is normal for Ash-based Phoenix apps. The 100MB target was set before the Ash ecosystem was added. Revisit if image size becomes a deploy bottleneck.

## Key Technical Decisions

1. **Compile before assets:** Phoenix LiveView 1.1 colocated hooks (`phoenix-colocated/haul`) are generated during `mix compile` and required by esbuild. The Dockerfile runs `mix compile` before `mix assets.deploy`.

2. **Elixir 1.19.3 vs 1.19.5:** Docker Hub's hexpm/elixir only has 1.19.3. The local system runs 1.19.5. This mismatch is minor (patch version) and the Docker image will use whatever is available. Update the ARG when 1.19.5 lands on Docker Hub.

3. **git in build stage:** Required because `heroicons` is fetched from GitHub as a sparse checkout dependency.

## Test Coverage

- **Build test:** `docker build -t haul:test .` succeeds ✅
- **Entrypoint test:** `docker run --rm haul:test ls bin/` shows expected scripts ✅
- **No integration test:** Running the container requires DATABASE_URL and SECRET_KEY_BASE. Full integration testing is a deploy concern (T-001-05/06).

## Open Concerns

1. **Image size exceeds target.** 278MB vs 100MB. The target should be revised upward or the Ash ecosystem should be acknowledged as the cause.

2. **Elixir version mismatch.** Local is 1.19.5, Docker Hub has 1.19.3. Not blocking but worth tracking. Update Dockerfile ARG when new image is published.

3. **No health check endpoint.** The Dockerfile doesn't expose a HEALTHCHECK. Fly.io handles health checks via its own configuration, so this is expected.

4. **No non-root user.** The runtime runs as root. Adding a non-root user is a security hardening step that could be added later.

5. **Windows .bat files.** `mix phx.gen.release` generated `.bat` files. Harmless but unnecessary in a Linux container.
