# T-001-04 Structure: Dockerfile

## Files Created

### `Dockerfile`
Three-stage multi-stage build:

**Stage 1: `deps`** (FROM hexpm/elixir)
- Set MIX_ENV=prod
- Install build-essential, git (for GitHub deps like heroicons)
- Copy `mix.exs`, `mix.lock`, `config/config.exs`, `config/prod.exs`
- `mix deps.get --only prod`
- `mix deps.compile`

**Stage 2: `build`** (FROM deps)
- Copy `lib/`, `assets/`, `priv/`, `config/runtime.exs`, `rel/`
- `mix assets.deploy` (fetches + runs tailwind and esbuild binaries)
- `mix compile`
- `mix release`

**Stage 3: `runtime`** (FROM debian:bookworm-slim)
- Install only `libstdc++6`, `openssl`, `libncurses5`, `locales`
- Set locale to en_US.UTF-8
- Copy release from build stage
- Set `PHX_SERVER=true`
- ENTRYPOINT: `bin/migrate_and_start`

### `rel/overlays/bin/migrate_and_start`
Shell script:
- Calls `bin/migrate` (runs Ecto migrations)
- `exec bin/server` (starts Phoenix with PHX_SERVER=true)
- Executable permissions (chmod +x)

### `rel/overlays/bin/migrate`
Generated by `mix phx.gen.release` — runs `Haul.Release.migrate()`.

### `rel/overlays/bin/server`
Generated by `mix phx.gen.release` — starts with PHX_SERVER=true.

### `lib/haul/release.ex`
Generated by `mix phx.gen.release` — provides `migrate/0` and `rollback/2` functions for running migrations without Mix.
